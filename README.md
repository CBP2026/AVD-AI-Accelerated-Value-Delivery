# Accelerated Value Delivery (AVD) Framework

A radical new approach to software delivery that compresses the build cycle from **months to weeks** — powered by AI, governed by clear gates, and evolved by the community.

## 🎯 What is AVD?

AVD is a framework designed for the era of AI-assisted development. It replaces the traditional 3–4 month software delivery cycle with a compressed, three-phase approach:

- **Phase 1 (Days 1–3)**: Rapid Discovery & MVP Build
- **Phase 2 (Days 4–5)**: Technical Review & Hardening  
- **Phase 3 (Ongoing)**: Continuous Evolution via Stakeholder Sandboxing

## 🚀 The Problem & Solution

**Traditional Agile:** 3–4 months to first value
- 2 weeks: Discovery
- 2 weeks: Design & planning
- 12–16 weeks: Development sprints
- 2 weeks: QA & release

**AVD:** 1 week to first value
- Direct discovery with business owner (no intermediaries)
- PM builds immediately with AI (no user stories)
- Embedded QA throughout (not post-sprint)
- Compressed technical gate (review by exception)

## 📋 The Three Phases

### Phase 1: Rapid Discovery & MVP Build (Days 1–3)
- PM sits directly with business owner
- Builds immediately with AI tools
- No user stories or wireframe handoffs
- Embedded QA catches issues in real time
- PO approves each feature daily with timestamp

### Phase 2: Technical Review & Hardening (Days 4–5)
- Automated security scanning
- Architecture validation against pattern library
- AI generates compliance documentation
- UX review (parallel track, not blocker)

### Phase 3: Continuous Evolution via Stakeholder Sandboxing (Ongoing)
- Certified stakeholders build in isolated sandboxes
- Community voting on features
- PO makes final call, explains rejections
- Rejected ideas archived with context (never deleted)
- Automated agents detect duplicates and monitor strategy shifts

## 📊 Key Metrics

| Metric | Traditional | AVD |
|--------|-------------|-----|
| **Time to First Value** | 3–4 months | 1 week |
| **Defect Escape Rate** | Post-sprint | Near-zero |
| **Feature to Production** | 2–4 weeks | Days |
| **PO Sign-off Cadence** | End of sprint | Daily |
| **Stakeholder Engagement** | PO-mediated | Direct |

## 🔐 Security & Governance

### Six Layers of Security
1. **Isolation** — Sandboxes fully isolated from production
2. **Synthetic Data** — Zero real personal information in sandboxes
3. **Automated Scanning** — All code scanned before community review
4. **Role-Based Permissions** — Auth, payments, exports locked to privileged roles
5. **Architecture Gate** — Phase 2 review required before production
6. **Audit Trails** — Complete traceability of who/what/when/why

### Governance Model

| Role | Traditional | AVD |
|------|-------------|-----|
| **Product Owner** | Backlog manager | Decision authority + daily reviewer |
| **Business Analyst** | Requirements intermediary | Eliminated |
| **Architect** | Multi-week review | 1–2 day AI-accelerated gate |
| **QA Tester** | Post-sprint testing | Real-time embedded quality |
| **UX Designer** | Upfront wireframing | Post-MVP compliance check |
| **PM** | Coordinator | Primary builder |

## ✅ First-Project Prerequisites

Before launching the first AVD project:
1. AI toolchain procured & IT-approved
2. PM certified on the toolchain
3. Pattern library baseline established
4. Deployment authority envelope agreed
5. AVD Coach identified
6. Sandbox infrastructure with SLA
7. Synthetic data layer (or in parallel development)
8. Programme-level sandbox cost pre-approved
9. Named backup PM with equivalent certification

## 🎓 Training & Certification

- Sandbox participation is **opt-in** (self-selection)
- Mandatory training before access covers:
  - Building with AI tools responsibly
  - Security awareness for sandboxes
  - Voting and sign-off process
  - Boundaries of what can be changed
- Experienced participants also acknowledge the new way of working
